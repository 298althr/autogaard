Strategic Bottlenecks (The "Red" List)

!

Persistence Instability (MVP Risk)

Reliance on localStorage. Any clear-cache event wipes owner compliance data and service history. Critical priority to migrate to PostgreSQL.

!

Static Health Scoring

"Beauty" and "Performance" scores are currently visual hard-codes. They do not dynamically calculate from actual service logs or diagnostic flags.

!

Missing "Vendor Orchestration"

The system expects workshop updates but has no "Partner Portal" to collect them. AutoGaard staff currently act as a manual bridge.

Implementation Roadmap (Do vs. Don't)

Immediate Actions (DO)

  • ✓

    Migrate to PG: Move all JSON structures to relational tables (vehicles_health, compliance_vault).

  • ✓

    Logic Unification: Map After-Scores to REAL service categories (e.g., Engine Scan + Oil Change = +5 Performance).

  • ✓

    Document S3 Sync: Replace base64 strings with remote file storage URLs for compliance docs.

Avoid (DON'T DO)

  • ✕

    Manual Pricing: Do not let workshops enter prices manually. Use the pre-set Calculators to maintain brand standard.

  • ✕

    Offline Documents: Do not start registration flows without digital vault verification. It breaks the audit trail.
